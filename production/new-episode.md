# new-episode.md — Воркфлоу нового эпизода

> Появилась тема — этот файл задаёт порядок: кто вступает, что передаёт дальше,
> где точки согласования с Nurlanом.
> Роли персонажей — в `channel/staff/`. Стиль — в `channel/style/`.

## Триггер
Новая тема (по контент-плану ИЛИ спонтанно «захотелось») = новый чат в проекте.
Один эпизод — один чат. Закончили — fresh session.

## Фаза 0 — Подготовка чата (Nurlan)
- [ ] Включить **Web search** и все нужные доступы (иначе Homer копает по памяти — риск по датам и деталям)
- [ ] Назвать тему
- [ ] Создать папку эпизода `episodes/epNNN-slug/` из `_template`

## Фаза 1 — Разведка (Homer) → `channel/staff/homer.md`
Homer берётся первым.
1. **Первичная разведка** — быстрый обзор темы по живым источникам.
2. **Уточняющие вопросы Nurlan** — что именно интересует, ракурс, насколько глубоко. Не угадывает.
3. **Глубокая проработка под угол подачи** — ищет зазор «версия vs реальность»,
   кандидатов в жгучий вопрос, нестыковки, фактуру, кандидатов на заход.
   Помечает: подтверждено / спорно / миф.

**Output:** досье → заполняет `episodes/epNNN-slug/brief.md`.

## 🔒 Точка согласования №1 (Nurlan)
Nurlan смотрит досье и кандидатов в жгучий вопрос → выбирает направление,
утверждает или заворачивает. **Без утверждения дальше не идём.**

## Фаза 2 — Сценарий (Nurlan + Claude)
Пишется по `channel/style/beats.md` + `channel/style/entry.md`.
Для удобства сначала пишется на русском языке. Но для сочности вставляемые английские идиомы/каламбуры даются сразу на английском.
То есть текст в меру может быть с английскими выражениями.
Голос — Arny, структура — 7 тактов, длина 20–30 минут.

## 🔒 Точка согласования №2 (Nurlan)
Nurlan читает черновик сценария → утверждает или возвращает на доработку.

## Фаза 3 — Перевод (Lingo) → `channel/staff/lingo.md`
Lingo переводит RU → EN транскреацией. Вшитые английские каламбуры переносит дословно.
Спорные места помечает `[LINGO: ...]`. Отдаёт готовый EN-слой в `script.md`.

## Фаза 4 — Правка языка (Ed) → `channel/staff/ed.md`
Ed проверяет: естественность американского английского, несуразицы,
конверсии и цифры, анахронизмы речи, повторы.
Отдаёт список замечаний — Nurlan правит сценарий.

## Фаза 5 — Критика структуры (Nolan) → `channel/staff/nolan.md`
Nolan проверяет: хук, 7 тактов, удержание, воду, видеоряд, тон, финал.
Вердикт: в эфир / подтянуть / пересобрать.

## Фаза 6 — Фактчекинг (Homer) → `channel/staff/homer.md`
Homer проверяет каждое утверждение сценария: источник, точность, причинность.
Помечает: подтверждено / спорно / миф. Тест: «переживёт враждебного эксперта?»

## Фаза 7 — Звуковая разметка (Murch) → `channel/staff/murch.md`

Murch берёт утверждённый EN-сценарий (после всех правок).

1. **Разметка подачи под ElevenLabs** — теги тональности, паузы, регистры по предложениям (правила — `production/voice-and-sound.md`).
2. **Музыкальный план по тактам** — стиль/настроение подложки, где входит/уходит (правила — `production/music.md`).
3. **Карта звуковых эффектов** — `[SAX]`, тишина, акценты.

Output: готовые файлы тактов `sound-ep-NNN-tN.md` (или `-pM` если такт длинный).

## 🔒 Точка согласования №3 (Nurlan)
Nurlan проверяет звуковые файлы → одобряет или возвращает Murch.
